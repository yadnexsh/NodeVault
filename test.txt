Excited to share something I've been quietly building for a while - NodeVault 🎉
It's my final project for the VSD Foundations course under GamutX, and honestly it turned out way bigger than I initially planned.

What is NodeVault?
NodeVault is a gizmo library manager that lives directly inside Nuke.
Think of it like a mini asset store built into your pipeline - artists can browse, preview & grab gizmos
without ever breaking their compositing flow. Contributors can submit new ones straight through the tool.

Why does it exist?
Every studio has the same silent problem - gizmos living in random folders,
shared over Zoom, copied manually, and half the team not even knowing they exist. There's no standard way to distribute, discover
or version control them. Artists waste time hunting for tools that someone else already built. NodeVault was built to fix exactly that.

How we solved it:
→ UUID-based folder per submission with structured JSON - no database needed, just the filesystem
→ Library rebuilds the grid live on every category click - new submissions appear instantly
→ Subscribe copies the gizmo straight into .nuke/NodeVault_User, keeping the full pipeline connected
→ Submit form validates every field before writing anything to disk - no broken entries ever reach the library

What it does:
→ Browse gizmos by category through a live tree-view panel (Deep, Image, Draw, Time, Channel, Filter)
→ View full details per gizmo - description, version, author, tagline, and external links
→ Open repo, website, and issue tracker links directly from inside the tool
→ Subscribe to any gizmo and it gets copied straight into your .nuke folder
→ Submit new gizmos with metadata, preview images, docs, and links - saved as structured JSON
→ Full validation on submit - catches every missing field before anything hits disk

Built entirely with:
→ Python + PySide2 (Qt) for the full desktop GUI
→ Nuke API for pipeline integration
→ JSON for structured metadata storage
→ shutil, uuid, os for file management across directories

This is v1 - and there's a lot more planned. More file type support, better search, deeper Nuke integration, and features that make it genuinely useful at
a studio scale. The foundation is solid, now it gets interesting.
